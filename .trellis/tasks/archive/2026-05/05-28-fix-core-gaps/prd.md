# PRD: 修复核心体验短板

## 背景

Agent 当前存在 3 个影响核心体验的问题，需按优先级修复并测试。

## 需求

### 1. 灵魂追问 bug（P0）

**问题**：`MAX_QUERY_ROUNDS=3`，但必问字段有 4 个（score、province、subject、family_background）。必问字段永远问不完，选问字段永远不会被触发。

**期望行为**：
- 4 个必问字段全部能被问到
- 之后可选问 1 个选问字段（target_city / risk_tolerance / career_goal）
- 用户可以跳过任何问题，跳过后应用默认值

**约束**：
- 不改变现有接口签名
- 保持张雪峰风格话术

### 2. 上下文裁剪（P0）

**问题**：聊天历史无限堆积，超过模型 context window 会报错。

**期望行为**：
- 历史消息超过阈值时自动裁剪
- 保留 system prompt + 最近 N 轮对话
- 裁剪对调用方透明

**约束**：
- 不引入外部依赖
- 裁剪策略简单可调（保留轮数可配置）

### 3. 会话持久化（P1）

**问题**：聊天记录纯内存（`sessions: dict`），重启丢失。

**期望行为**：
- 聊天记录持久化到 SQLite
- 重启后可恢复历史会话
- 按 session_id 读写

**约束**：
- 使用项目已有的 SQLAlchemy
- 不改变现有 API 接口
- SQLite 文件路径可配置

## 验收标准

- [ ] 灵魂追问：4 个必问字段全部能被问到
- [ ] 灵魂追问：选问字段在必问完成后能被触发
- [ ] 上下文裁剪：50+ 轮对话不报错
- [ ] 会话持久化：重启后能恢复历史
- [ ] 所有修改有对应测试
- [ ] 现有测试全部通过
