# PRD: 功能优化

## 背景

按优先级逐项优化，每项独立可验证。

---

## P1-1: search_policy stub 清理

**问题**：`search_policy` 工具永远返回 `"not_implemented"`，浪费 token 且 LLM 可能反复调用。

**方案**：从工具注册中移除 `search_policy`，或改为返回友好提示 `"该功能暂未开放"`。

**验收**：LLM 不再调用 `search_policy`；若仍调用，返回友好提示而非 JSON stub。

---

## P1-2: LLM 调用重试 + 降级

**问题**：OpenAI 偶尔 429/500，当前直接报错给用户。

**方案**：
- 在 `AgentCore.chat` 和 `chat_stream` 中添加指数退避重试（最多 2 次，间隔 1s/2s）
- 仅对 429/500/502/503 重试，其他错误直接抛出
- 重试耗尽后返回友好错误提示

**验收**：模拟 429 响应时自动重试；重试耗尽后用户看到友好提示。

---

## P1-3: 前端历史会话列表 + 恢复

**问题**：后端已持久化会话，但前端没有会话列表，无法恢复历史。

**方案**：
- 后端新增 `GET /sessions` 列表接口（返回最近 20 个会话）
- 前端 Portal 页添加"历史会话"入口
- 点击会话跳转到 Chat 页并传入 session_id
- Chat 页加载时自动拉取历史消息

**验收**：重启后能看到历史会话列表；点击后恢复完整对话。

---

## P2-1: 工具结果缓存

**问题**：相同查询重复调 DB，浪费资源。

**方案**：
- 在 `tool_registry.dispatch` 层添加简单的 TTL 缓存（内存 dict + 过期时间）
- 缓存 key = `func_name + sorted(args)`
- 默认 TTL 5 分钟，可配置

**验收**：相同参数的连续调用第二次命中缓存（日志可见）。

---

## P2-2: 对话导出

**问题**：用户想保存咨询结果。

**方案**：
- 后端新增 `GET /session/{id}/export` 返回 Markdown 格式的对话记录
- 前端 Chat 页添加"导出"按钮，触发下载

**验收**：点击导出可下载 `.md` 文件，包含完整对话。

---

## P3-1: 反馈机制

**问题**：前端已有 `FeedbackRating` 组件调用 `/api/v1/feedback`，但后端没有该接口。

**方案**：
- 后端新增 `POST /api/v1/feedback` 接口 + SQLite 存储
- 后端新增 `GET /api/v1/feedback/stats` 统计接口
- 确保前端组件能正常调用

**验收**：前端点赞/点踩能成功提交；统计接口返回正确数据。
